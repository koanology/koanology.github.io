Tricks

Some programming tricks exist which can be faster than the intuitive way of doing things.

Mapping

The goal of this technique is to avoid branching. The idea is to find a way to map the input to the correct output using operations other than branching. This usually results in more CPU ops but it doesn't slow down the pipeline (whereas branching may).

Absolute Value Mapping

The absolute value function results in 3 CPU (integer) operations (unless there is a single hardware op). Branching may slow down a program because execution may not be predictable (i.e. it slows down the pipeline). By using the absolute value function we can make code execution predictable and potentially faster.

Dynamic Difficulty Adjustment

Players will likely quit a game at some point if it becomes too hard or too easy. Dynamically adjusting the difficulty not only involves predicting the psyche of pro gamers but casuals gamers as well.

Leaky Spawn System

A map can only fit so many units on it at one time. To simulate a map that holds more units than it actually can, keep respawning destroyed units as if they were coming from just outside the map or just outside of view.
