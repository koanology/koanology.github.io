Brands

RDBMS

Relational database management systems are based on math (specifically set theory). A properly constructed schema ensures the integrity of whatever data it stores. If you want your data to be "bulletproof" then look no further.

Oracle

An expensive product although it has had tons of money and decades of research put into it. Basically a name brand product. For historical reasons, Oracle pretends NULL and empty strings are the same thing. Thankfully, Oracle calls this hickup VARCHAR2, which helps avoid issues when working with systems that are more compliant with SQL standards. The company has tried pushing products into other markets but is only a major contender in the RDBMS segment (not that that matters considering their revenue).

DB2

An IBM product that lives up to Big Blue's reputation. Cheaper than Oracle and actually handles NULL strings properly.

Postgres

A free product that can go toe-to-toe with DB2/Oracle. Postgres provides great OLTP support with ACID transactions and strives for compliance with SQL standards. Other database vendors may provide more OLAP capabilities although it's usually more cost effective to just use other software (e.g. NoSQL) rather than pledge your allegiance to a huge company. Also, if you're denormalizing data you mine as well switch to a NoSQL system.

SQLite

A free database engineered to work like Postgres embedded directly into your application. Not recommended for OLTP (you need a client/server database for that) but works well for OLAP.

NoSQL

Not all data needs to be "bulletproof" and such level of data integrity comes at a price. RDBMS systems are not good at scaling horizontally because there are so many checks in place to make sure all the data is correct all the time. This is where NoSQL comes in.

Programming

Bash

The standard for *NIX scripting. Just glue a bunch of commands together and voila!

C

Compiles on practically any hardware platform. Most operating systems are written in it.

JavaScript

The defacto programming language for the web. Most sites use it since it's practically required. For simple websites jQuery is enough but anything involving more than a few hundred lines of code would be better served using Dojo. It's like comparing Legos to a 3D printer.

Lisp

Expanded the art of AI research decades ago and still influences other languages to this day.

Lua

The defacto language for embedding scripts into other languages. It's like a scripting version of C but with a much easier learning curve (and you can call C code from it too).

SQL

The defacto language for processing data.

Household Name

Apple

Good mobile devices. Otherwise, overpriced namebrand.

Google

Their specs usually lack in something. Their Nexus line could be better although decent products all around.

Microsoft

Useful for games and certain software that requires Windows. Otherwise not worth the money. Although, Microsoft Research is on another level compared to the rest of the company.

Samsung

Basically replaced Nokia as the king of mobile. Their Note line was good up until the 4 and then they screwed up every version after that. Their chips and screens are included in most benchmarks when comparing products. They've really solidified their position in multiple markets.

OS

An operating system is what most software is coded against (sometimes software is written directly in assembly).

Arch

A highly customizable Linux distro with tons of documentation.

BSD

A locked down *NIX with plenty of documentation. It's very organized and well engineered, which affects its compatibility with certain hardware/software. Basically it's not as good as Linux for desktop applications. Otherwise it's great for network appliances and secure servers.

Redhat

The industry standard for Linux.
